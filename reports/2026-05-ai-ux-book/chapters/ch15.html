15.1体检方法论 · 4 步流程

这次体检按 4 步流程做,每条诫都过一遍。

  1. 对照诫的「怎么落地」标准,描述千图AI 当前状态
  2. 打分 0-10(0 = 完全没做 / 10 = 行业标杆)
  3. 定位关键缺口(具体场景 / 具体页面)
  4. 给出改进建议(可落地动作)

分数仅是「内部相对参考」,不是行业横评。同诫不同产品的对比见 Part III。

15.3优势分析 · 哪几诫做得好

先看强项,这是产品的护城河,要继续保持。

优势 1 · 第 3 诫(逃生通道)· 7/10

千图AI 的核心生成失败时,用户能「重生 / 改 prompt / 改风格 / 下载原图自己改」——多条退路。这比 Rabbit R1 那种「卡住只能等」的产品好得多。

但可以更好:加上「局部重绘」(inpaint)入口的可见度——目前用户喜欢 70% 的图时,主流操作是「重生」而不是「改剩下 30%」。这就让 ROI 不优化。

优势 2 · 第 4 诫(预览态)· 7/10

生成的图天然在「预览态」——用户主动下载才算 commit。这个好习惯是图像产品的天然优势。

但可以更好:涉及「分享到社区 / 发布为模板 / 提交评审」等公开行为时,缺「二次确认 + 公开影响提示」。

优势 3 · 第 8 诫(撤销底线)· 7/10

历史记录可查,生成不会「消失」。

但可以更好:历史是「平铺列表」形态——用户基于第 3 张图 refine 得到第 5 张,这种派生关系不可见。学 Krea 的 canvas 派生树。

优势 4 · 第 12 诫(让人更聪明)· 7/10

没有「打卡 / 连续签到 / 沉迷诱导」机制,健康度高。

但可以更好:加「prompt 教学」——当用户写出好结果时,主动展示「这张图为什么生成得好(关键 prompt 词分析)」,让用户学到能迁移的技巧。

15.4缺口分析 · 哪几诫缺口最大

下面 5 条是 ROI 最高的改进项,推荐优先做。

缺口 1 · 第 1 诫(信任)· 5/10

✕ 当前问题
  • 生成图旁边没有「这张图相对你 prompt 的契合度评估」
  • 没有明示当前用的模型版本(用户不知道用的是即梦 5.0 还是 4.0)
  • 多张生成结果之间,没有「我们推荐选这张」的引导
✓ 改进建议
  1. 给每张图标「AI 自评分」(prompt 匹配度 / 美观度 / 安全度 三档),让用户秒判
  2. 明示模型版本(底部小字「由 千图AI v3.2 生成」)
  3. 多图推荐(在 4 张备选里给「我们更推荐这张」的小标记 + 理由)

缺口 2 · 第 2 诫(错误兜底)· 5/10

✕ 当前问题
  • 「生成失败」提示过于通用,用户不知道是 GPU 排队 / prompt 触发安全 / 模型本身波动
  • 内容安全拦截时,不告诉用户具体触发了哪个关键词
  • 失败 3 次后,没有「升级提示」(看起来不是临时问题,试试 X)
✓ 改进建议
  1. 错误分类提示(用 5 类错误模板:系统级 / 输出级 / 理解级 / 边界级 / 时间级)
  2. 安全拦截具体到关键词(在不暴露内部规则的前提下,给用户改的方向)
  3. 失败 3 次升级(主动给「试试改成 [建议 prompt]」)

缺口 3 · 第 5 诫(prompt 体验)· 6/10

✕ 当前问题
  • placeholder 通用化(「请输入您的需求」),不引导中文用户
  • 缺「中文场景模板库」(对比即梦「电商海报 / 公众号封面 / 抖音封面」等模板入口)
  • prompt 失败后,缺「自动改写建议」
✓ 改进建议
  1. 场景化 placeholder(「试试:为新年活动设计一张红金主调的海报」)
  2. 建中文场景模板库(电商主图 / 公众号封面 / 朋友圈背景 / PPT 配图 / 微信表情等),用户选场景再填关键字
  3. prompt 改写助手(用户结果不满意时,主动给「试着加上 X 描述」)

缺口 4 · 第 9 诫(透明度)· 5/10

✕ 当前问题
  • 用户不知道当前用的什么模型 / 版本 / 参数
  • 「AI 改写 prompt」(如果有的话)是黑盒,用户看不到 AI 把自己的 prompt 改成了什么
  • 缺「为什么这张图长这样」的可解释入口
✓ 改进建议
  1. 结果旁边展示「生成参数」(模型 + 风格 + seed + guidance),可点开看详情
  2. 暴露 prompt 改写(如果 AI 内部改写了用户 prompt,展示「原始 prompt → 实际使用的 prompt」)
  3. 可复现性(用户能复制「完整生成参数」用来重生 / 给别人复现)

缺口 5 · 第 10 诫(上下文资产)· 4/10

✕ 当前问题
  • AI 不学习用户偏好——用户每次都要重写「我要写实风格」等基础偏好
  • 项目化(Projects)概念缺失——用户做同一个项目的不同图,系统不知道这是一组
  • 跨设备 / 跨账号的「用户偏好」无法迁移
✓ 改进建议
  1. 加 Memory / Custom Instructions(用户能设置「我永远要写实风格 / 永远不要插画风 / 永远生成 4 张」等全局偏好)
  2. 加 Projects 概念(类似 ChatGPT Projects / Claude Projects——一个项目下的所有生成共享上下文)
  3. 偏好可审计(用户能看到 AI 记了什么偏好,能逐条删 / 改)

15.5具体场景体检 · 4 个核心流程

把抽象诫拉到具体场景,看千图AI 的 4 个核心流程各自有什么 UX 问题。

场景 1 · 「生成新图」流程

  • 当前流程:打开生成页 → 写 prompt → 选风格 → 点生成 → 等结果 → 满意下载 / 不满意重生
  • 核心问题:每次都从「空白」开始,缺场景模板入口
  • 改进:首屏给 8-12 个场景模板卡片(海报 / 封面 / 配图 / 头像等),用户选场景后填关键字

场景 2 · 「改已有图」流程

  • 当前流程:不明显(用户多倾向重生)
  • 核心问题:Inpaint / 局部重绘的入口不显眼,用户不知道有这个能力
  • 改进:每张生成图下方加「局部修改」按钮(跟「重新生成」同等显眼),用户拖框选区域 + 描述改什么

场景 3 · 「历史回查」流程

  • 当前流程:历史列表 → 找到某张 → 下载 / 再生成
  • 核心问题:平铺列表难找,缺「派生关系树」
  • 改进:历史改成「画布」形态(类似 Krea / Figma)——每张图是节点,基于某张图改出来的图是它的子节点,可视化派生

场景 4 · 「失败重试」流程

  • 当前流程:看到失败提示 → 点重试 → 失败 → 再点重试
  • 核心问题:无升级 / 无诊断 / 无替代建议
  • 改进:失败时给具体原因 + 3 个建议(如「prompt 触发了 X 关键词,试试改成 Y / 当前 GPU 排队,等 N 秒 / 换风格 Z 可能更稳定」)

15.6体检小结

千图AI 在 12 诫上的整体得分 ~ 6.5/10——核心生成能力扎实,但5 条主要缺口(信任 / 错误兜底 / prompt 体验 / 透明度 / 上下文资产)是改进 ROI 最高的方向。

4 个具体场景(新图 / 改图 / 历史 / 失败)各自有可立刻动手的改进点。这些改进的具体顺序、时间、所需投入,见下一章千图AI 90/180/365 天改进路线图。
