LLM 是个什么东西 A One-Word-at-a-Time Parrot

在讲产品、讲工程之前,先讲清楚最基础的一个问题:你每天面对的 ChatGPT / Claude,内部到底在干什么?这一章不教你公式,只用 4 个性质和大量类比,让你能跟工程师对话、能预判 LLM 的脾气、能在它幻觉时立刻反应过来。

▼ ELEVATOR PITCH

LLM 是一个一个字一个字蹦的鹦鹉,它做的事只有一件:看完前面的所有字,预测下一个字。它不"知道"任何事——只是把训练里读过的几万亿字压缩成了概率分布。理解了这一点,你就能解释它的所有怪癖:为什么它会一本正经胡说八道,为什么它能写诗但不会做小学算术,为什么换一个 prompt 行为就完全不同,为什么"它说它确定"这句话本身就是一种幻觉。

故事 / 2023 年 5 月 · 纽约联邦法院

2023 年 5 月,纽约执业 30 多年的律师 Steven Schwartz 接了一个普通的人身伤害案。他要给法庭提交一份摘要,引用过往判例支撑当事人的诉求。他用 ChatGPT 帮忙找判例,几分钟就拿到了一份漂亮的清单——案件名称、卷宗编号、判决要点,看上去专业极了。

他甚至追问 ChatGPT:"这些案件是真的吗?"ChatGPT 答:"是的,这些都是真实的案件,可以在法律数据库查证。"于是他把清单原样交了上去。

几天后,法院告诉他:这 6 个案件,没有一个是真的。案件名是编的,卷宗号是编的,引用的法官原话也是编的——但所有内容都看起来像真的,因为 ChatGPT 学过无数真实判例的写法,它知道一份正经的判例摘要应该长什么样。它只是不知道它列的那 6 个具体的案例不存在。

Schwartz 被罚款 5000 美元,从此成为"AI 律师事故"的代表人物。但故事的真正价值,在于它把 LLM 的一个核心真相摆到了公众面前——它输出的"自信"和"事实"完全是两件事。要理解这一点,得先看 LLM 在内部到底干了什么。

1.1第一个性质:下一词元预测 · Next Token Prediction

把 LLM 的所有能力剥到最底,只剩一件事:给定一段已有的文字,预测下一个最可能的字(或字片段)是什么。仅此而已。

它不"理解"问题,不"思考"答案——它只是看着前面一长串字,从词典里挑下一个最该出现的字,然后把这个字加到末尾、再挑下一个,如此循环,直到生成一个结束标记。

这听起来太简陋,简陋到让人怀疑:"那它怎么能写出读起来像人话的句子?"——答案是规模。当一个模型学过几万亿字的人类文本,它见过几乎所有的"前文 + 后文"组合;它不需要真的理解语义,只需要把"在这种前文之后,下一个字通常是 X"这种统计规律编码进去。

一个具体的演示

假设你输入:"今天天气真"。LLM 内部不是"想"出一个答案,而是给每个可能的下一字算一个概率:

TOKEN 1
「好」
P ≈ 0.42 · 训练里最常见
TOKEN 2
「不错」
P ≈ 0.18 · 也很常见
TOKEN 3
「冷」
P ≈ 0.09 · 季节相关
TOKEN N
「⋯」
几万个可能,概率递减

它按这个概率分布采样出一个字(比如选了「好」),把它接到原文后面变成 "今天天气真好",然后再用这一整段重新算下一个字——可能是「,」或「。」或「啊」。如此循环,一字一字蹦出来。

i 关键观察 · 它没有"全局规划"

LLM 写一段 500 字回答时,它不是先想好整段、再一字字写出来——它是真的一字一字往后蹦,每个字都基于前面已经蹦出来的所有字重新算概率。这意味着:它写第 50 个字时,完全不知道第 200 个字会是什么。它的"逻辑连贯"完全来自局部的统计规律,不来自全局规划。

这一性质带来的三个直接后果

后果一 · 它能生成流畅的废话。流畅性是它训练目标的唯一指标——"下一个字概率最大的那个"通常就是流畅的。真实性、准确性、新颖性,都不在它的目标函数里。所以它默认就是个"流畅生成器",真实不真实是意外的副产品。

后果二 · 同样的输入,每次输出可以不同。从概率分布里采样,本身就有随机性。Anthropic 的 API 有个 temperature 参数控制这个随机性:temperature=0 时永远挑概率最大的那个字(确定性输出),temperature=1 时按概率分布原样采样(发散输出),temperature=2 时几乎乱选(混乱输出)。理解了这一点,你就懂为什么同一个 prompt 会给你不同答案。

后果三 · 它对"前文"极度敏感。前面多一句话、少一句话、换一个语气词,都会让概率分布完全重算。这就是 prompt engineering 能存在的根本原因——你不是在"指挥"它,你是在塑造它的概率分布。

✕ 常见误解 · 「LLM 在思考 / 推理 / 理解」

这三个词都是人类经验的投影,LLM 内部没有任何对应物。Claude 写出"让我一步步分析这个问题"不是因为它真在分析,是因为在训练语料里,这种句式之后通常会出现一个看起来像分析过程的文本。Chain-of-Thought 之所以有效,不是因为我们"教会了它思考",是因为我们把它的局部概率分布拉到了"显式推理文本"的轨道上,让它沿着这条轨道一字一字蹦,最终结果更可能正确。机制是统计,效果像思考——这是 LLM 时代最容易混淆的一组概念。

1.2第二个性质:知识 · Knowledge

LLM 在预测下一个字时,需要某种"经验"来知道哪些字更可能出现。这种经验,我们叫它知识——但它跟人类记忆里的知识,长得不一样。

人类记忆是显式的——你可以回忆起"水的化学式是 H₂O",这条信息存在你脑子的某个具体位置,你能取出来、能验证、能解释来源。LLM 的"知识"则是隐式压缩进了几千亿个参数——没有一个特定参数存"H₂O",这条事实是分散在整个网络里、以某种统计模式涌现出来的。

这种知识形态的三个怪癖

怪癖一 · 它有"截止日期"

LLM 是用某个时间点之前的语料训练的。Claude Opus 4.7 知道 2026 年 1 月之前的事,但 2026 年 3 月发生的新闻它不知道——除非你在 prompt 里告诉它。"知识截止日"是 LLM 内置的根本性限制,不是 bug。

怪癖二 · 它"知道"和"自信"是两件事

LLM 没有一个内部信号能告诉它"我对这条信息有 90% 把握"。它的输出概率高,只意味着"在这种前文下,这个回答在训练语料里最常见"——这跟事实正确与否完全不相关。所以当它说"我确定"时,这句话本身就是一个被概率选出来的字串,不是它真的"确定"。

怪癖三 · 罕见事实的"幻觉率"远高于常见事实

问 LLM 美国第一任总统是谁,它准确率近 100%——因为训练语料里出现过几百万次"乔治·华盛顿"。问它某个 1962 年俄罗斯小诗人的某首诗第三段,它会给你编一段——因为相关语料太稀疏,它没有可靠的统计基础,但它仍然会生成看起来像那首诗的字。Schwartz 律师案就是这个怪癖的极端版本:法律判例数据很多,但具体到某个特定案号的精确细节属于"长尾",LLM 用统计填空填得很流畅,但填的是假数据。

怪癖四 · 它对自己的知识范围没有自知之明

问人类"你知道俄罗斯历史吗?",人类会说"我不太懂,只知道一些大事件"。问 LLM 同样的问题,它会说"我对俄罗斯历史有相当深入的了解",然后真的开始讲——讲的全是它训练里见过的内容,但它分不清自己见得多还是见得少。让 LLM 自我评估"我对这个话题有多熟悉",答案高度不可靠。

i 一个有用的类比 · 「一个读过整个图书馆的健忘症患者」

想象一个人,他通读过几万亿字的人类知识,但他没有任何能回忆"在哪本书上看过这件事"的能力——他只能基于"我读过的所有书里,这种问题通常这么回答"来生成答案。他记得"模式",不记得"出处"。这个类比能解释 LLM 几乎所有的知识层面行为——为什么它能写得像专家,为什么它无法给真实引用,为什么它对自己读没读过某本书完全没把握。

1.3第三个性质:工作记忆 · Working Memory · Context Window

如果说"知识"是 LLM 长期记忆里压缩的几万亿字训练语料,那"工作记忆"就是它当下能看到的、正在用来预测下一个字的那段文字——也就是 上下文窗口(Context Window)。

这一概念决定了 LLM 几乎所有的对话级行为。理解了上下文窗口,你就懂了为什么 Claude 会"忘记"开头说过的话、为什么塞太多信息反而效果变差、为什么 Anthropic 反复强调"system prompt"的重要性。

上下文窗口的三个要点

要点一 · 它是有上限的。Claude Opus 4.7 在 2026 年是 200K tokens(约 15 万中文字),Sonnet 4.6 也是 200K,Haiku 4.5 是 200K。这个上限是硬性的——超过的部分会被截掉,LLM 看不到。1M context 模式是 Claude Opus 4.7 提供的特殊接入,把窗口扩到 100 万 tokens(约 75 万中文字),但成本和延迟都显著上升。

要点二 · "看得到"不等于"用得好"。即使 200K tokens 都在窗口里,LLM 也不是均匀使用所有内容——它对开头和结尾的内容更敏感(LLM 研究者管这叫"lost-in-the-middle"现象,Liu 2023)。如果你把关键指令埋在 10 万字的中间,它大概率不会注意。所以好的 prompt 工程是把最重要的放最前面或最后面。

要点三 · 它没有"长期记忆"。一次对话结束后,上下文窗口清空——下次对话 Claude 完全不记得你是谁、上次聊过什么。Anthropic 的 Projects(项目)和 Memory(记忆)功能,本质是把上次的关键信息自动塞回这次的上下文窗口,而不是 LLM 真的"记住了"。这是个工程实现,不是模型能力。

✕ 常见误解 · 「上下文越大越好」

不是。上下文窗口里塞的每个 token 都要算成本和延迟,而且 LLM 在大窗口下注意力会变稀(同样的关键指令,在 5K context 里它会专注;塞进 100K context 里它就可能漏看)。200K 窗口的正确姿势是:把任务真正需要的信息塞进去,而不是把所有相关资料都倒进去。塞进去的每个 token 都在跟你的指令竞争 LLM 的注意力。

一个反直觉的实验

有研究 (Liu 2023) 拿同一个问答任务,把答案藏在不同位置:开头、中间、结尾,再看模型答对的概率。规律很干脆——答案藏在开头或结尾,准确率都在八成上下;一旦塞进正中间那段,准确率掉到一半左右。

答案位置准确率判断
窗口开头 10%~85%注意力强
窗口中间 40-60%~50%注意力最弱 — 这就是 lost-in-the-middle
窗口结尾 10%~80%注意力强

这条规律对 prompt 设计有直接影响:关键指令和重要上下文,放窗口开头或结尾,不要埋在中间。Anthropic 官方建议把 system prompt 放开头,把"最重要的问题"放用户消息的末尾——就是利用这个规律。

1.4第四个性质:可操控性 · Steerability

前面三个性质决定了 LLM能做什么。这第四个性质决定了它能被你拽到哪个方向上做——也就是 prompt engineering 之所以能存在的理论基础。

LLM 的输出对前文极度敏感。前文里多一句"你是一位严谨的科学家",它就会偏向使用严谨的措辞;多一句"用一个 10 岁孩子能懂的方式回答",它就会切换到比喻和简单类比。这种敏感性,就是可操控性。

可操控性的四个维度

ROLE
角色
"你是一位…"塑造输出风格
FORMAT
格式
"用 JSON / 用列表 / 写 200 字"
EXAMPLE
范例
给几个示例,它会模仿
CONSTRAINT
约束
"不要…/必须…/只能…"

能拽动它的把手归起来就四个,记住就够用了。一是角色,给它一个身份;二是格式,直接点名要列表、要 JSON、还是只写两百字;三是范例,给几个示范让它照着模仿;四是约束,用「不要、必须、只能」划出边界。四个把手拧的都是同一件事——它对前文的那点敏感。

这四个维度对应 Anthropic 反复强调的"prompt 工程五原则"(在 第 9 章 详讲)的底层。所有 prompt 技巧——few-shot learning、role prompting、Chain-of-Thought、XML 标签——本质都是在拉扯 LLM 的概率分布,让它沿你要的轨道蹦字。

i Anthropic 的特殊设计:Constitutional AI

Anthropic 在训练 Claude 时,用了一种叫 Constitutional AI(宪法 AI,Bai 2022)的方法——让 Claude 在生成时持续自我评估是否符合一组"宪法原则"(比如有用、诚实、无害)。这导致 Claude 的可操控性带着一个底层偏置:它默认就有"我应该有用 + 诚实 + 无害"的倾向,你 prompt 时可以拉它的风格,但拉不动它的底线。这是 Anthropic 跟其他厂商最不一样的设计选择——你能感觉到 Claude 比 GPT 更"克制"、更愿意说"我不知道",就来自这一层。

可操控性也有极限

不是所有事 prompt 都能拽。下面是几条 prompt拽不动的事:

  • 训练里完全没见过的格式 — 一种你自创的全新标记语言,LLM 第一次见,prompt 怎么写都难以稳定输出。
  • 需要它"真的知道"的事实 — 你 prompt 它"列 10 个 1842 年某小县城的县令",它会编 10 个,因为训练里这条信息根本没存。Prompt 不能凭空给它知识。
  • 计算密集型推理 — 让 LLM 直接做 8 位数乘法,它会算错(因为它一个 token 一个 token 蹦,没有计算器机制)。Chain-of-Thought 部分缓解但不解决——彻底解决要给它一个 code execution 工具(在 第 8 章 讲)。
  • 违反它训练里的安全底线 — Anthropic 的 Constitutional AI 让 Claude 在某些话题上"卡死",prompt 怎么诱导都不松口。这是设计选择,不是 bug。

1.5四个性质冲突时:幻觉的来源 · When Properties Collide

单独看每个性质都没什么问题。LLM 的所有"怪事",都发生在四个性质相互冲突的时候——特别是幻觉。

把 Schwartz 律师案放进四性质框架,你就能看清幻觉怎么生成的:

性质在 Schwartz 案里的作用
Next Token Prediction问"列举支撑 XX 的判例",LLM 必须生成"看起来像判例清单"的文本——它的目标就是流畅,不是真实
Knowledge判例写法的统计模式在训练里出现过几百万次,但具体某个真实案号 + 真实判官 + 真实判词属于稀疏长尾,统计基础薄弱
Working Memory上下文窗口里没有任何"必须从真实数据库检索"的指令,LLM 没办法拒绝编造
Steerability问"这些是真的吗",它会按训练里"被问真实性时的常见回答"来——通常是肯定的,因为人类问这种问题时多半也希望听到肯定

四个性质在这桩案子里各按了一格。预测下一词元逼它生成一份读着像判例清单的东西,目标是流畅、不是真实;知识那头,判例的写法见过几百万次,可某个具体案号、具体判官、具体判词,落进了稀疏长尾,根基很薄;工作记忆里又没人下过「必须去真实数据库查证」这道指令,它便没理由拒绝编;最后被问到「这些是真的吗」,可操控性让它顺着人最想听的那个答案点了头。

四个性质合起来,完美再现了那场事故。没有任何一个性质单独导致幻觉,但四个加起来,幻觉就是这种系统的结构性产出。理解到这一层,你就懂了:防幻觉不能靠 LLM "自己变聪明",得靠系统设计上的多层防御(第 4 章 专门讲)。

✕ 一个反直觉但重要的事实

更强的模型,幻觉不会消失,只会变得更难发现。GPT-3.5 编的引用,你一眼就觉得别扭;Claude Opus 4.7 编的引用,作者名、年份、期刊都符合那个领域的真实分布,你需要去数据库里查才知道是假的。模型越强,幻觉越像真的——这是 LLM 这套机制下,你必须接受的代价。所以幻觉防御要靠外部验证机制(RAG / Citations / 工具调用),不能靠"模型变好"。

1.6这一章给你的判断武器

读完这章,合上,你应该带走以下几个能直接用的判断武器。

✓ 武器一 · 看到 LLM 输出"我确定 / 这是事实 / 已验证"时,这些字本身不是证据

这些字是从概率分布里采样出来的,跟它真不真实无关。要把它当成"LLM 在说这话",而不是"LLM 知道这件事"。

✓ 武器二 · 罕见事实的"自信回答"应该提高警惕

问主流知识(美国第一任总统),LLM 准确率高。问罕见事实(具体某个判例的某段引言、某本小众书的某句话、某个非常具体的统计数据),LLM 大概率在编——即使它说得很自信。

✓ 武器三 · 关键指令放窗口开头或结尾,不要埋中间

200K context 的中间 40-60% 区域注意力最弱(lost-in-the-middle)。要 LLM 注意的事,放最前面或最后面。这一条对所有用 Claude 处理长文档的人都直接适用。

下一章我们换个尺度——LLM 的"四个性质"是静态视角,但 Anthropic 真正在推的是动态形态:把 LLM 嵌进一个能自主感知-行动-验证的循环里,变成 Agent。Agent 跟 Chat 不是一个东西,它们对你的合作方式完全不同——这个差别,是 Claude Code 之所以让程序员效率翻倍的根本原因。
